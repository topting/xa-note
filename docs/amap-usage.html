高德地图多版本兼容运行

Sunny.D

# 背景

系统已经引入 高德地图 V1.4.15 版本，因业务需求，需要另外引入 V2.0 版本

# 遇到的问题

通过 JS API 引入高版本地图后，使用低版本的地图时，出现控制台报错。

  • 大致原因：v1.4.15 的地图是通过 <script> 标签引入，而 v2.0 则是通过 JS API 引入。不同的页面加载时，每次都重新加载地图，导致 AMap 对象被不断重写。
  • 导致的结果：当低版本页面需要加载地图时，可能当前只有高版本的 AMap对象，则会出现插件等引用错误。高版本页面，亦是如此。

# 思路

  • 缓存不同的 AMap 对象，以及 _jsload_ 方法。当页面加载需要的版本时，读取对应版本的对象、方法

# 实现

  • 入口 index.html
<!-- v1.4.15-->
<script>
    window.AMap1 = window.AMap
    window._jsload_1 = window._jsload_
</script>

  • 组件页面
// AMap 1.4.15
mounted () {
    window.AMap = window.AMap1 || null
    window._jsload_ = window._jsload_1 || null
}

// AMap 2.0
// 需要先引入 AMapLoader，再存储 _jsload_ 与 AMap
import AMapLoader from '@amap/amap-jsapi-loader'


mounted () {
    window.AMap = window.AMap2 || null
    window._jsload_ = window._jsload_2
    AMapLoader.load({
        key: '',
        version: '2.0',
        plugins: ['AMap.Scale', 'AMap.OverView', 'AMap.ToolBar', 'AMap.MapType', 'AMap.PolyEditor', 'AMap.PolygonEditor']
    }).then((AMap) => {
        window.AMap2 = AMap
        window._jsload_2 = window._jsload_
    }).catch(e => {
        console.log(e)
    })
}
